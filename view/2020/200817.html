    小程序中二维码的生成和保存

    wangshanjian 2020-08-17 vue uni-app 小程序

    由于公司项目需要生成二维码并保存到相册中，其中经历了对接后端API 接口生成二维码，后台传给前台base64位的图片，再经过解密，用户相册授权，拒绝授权的二次提示，最后的保存到手机的过程。

    # 1.生成二维码

    由于项目需要二维码需要携带参数，我们选择适用于使用数量极多的场景。接口地址：

    https://api.weixin.qq.com/wxa/getwxacodeunlimit?access_token=ACCESS_TOKEN

    # 通过scene参数来给页面添加参数

    一般我们主要常用的参数是：scene（如果需要页面参数）、page和width。 page是页面地址，例如：'pages/index'。pages前面不能有斜杠 scene是参数，为字符串。比如要传入一个用户id=1234，要根据这个用户id来给当前页面返回不同的内容，那么scene参数就可以写成"1234"，多个参数按一定规则分开，如&符号，第二个参数是recommendId=123则可以这样写"1234&123"。

    getWxQrCode() {
	let id = uni.getStorageSync('userId')
	this.$api.user.userInfoGetWxQRCode({
		scene: id,
		page: "pages/index/jobCardDetail"
	}).then(res => {
		if (res.statusCode === 200) {
			this.qrCodeUrl = res.data.data
		}
	})
},

    # 保存图片

    使用writeFile及saveImageToPhotosAlbumAPI保存至相册

    var imgSrc =  this.data.imgData;         //base64编码
var save = wx.getFileSystemManager();
var number = Math.random();
save.writeFile({
    filePath: wx.env.USER_DATA_PATH + '/pic' + number + '.png',
    data: imgSrc,
    encoding: 'base64',
    success: res => {
        wx.saveImageToPhotosAlbum({
            filePath: wx.env.USER_DATA_PATH + '/pic' + number + '.png',
            success: function (res) {
                wx.showToast({
                title: '保存成功',
                })
            },
            fail: function (err) {
                console.log(err)
            }
        })
        console.log(res)
    }, fail: err => {
        console.log(err)
    }
})

    # 3.二次授权问题

    我需要小程序用户授权保存图片至手机相册权限，当用户点击授权弹框的拒绝时，再次使用下载功能需要再次弹出授权页面，但是调用wx.authorize方法会返回错误信息errMsg: "authorize:fail:auth deny"，指用户已经拒绝授权，不会在弹出授权页面。

    let sm = wx.getFileSystemManager();
let that = this
sm.writeFile({
    filePath: wx.env.USER_DATA_PATH + '/zcbaiy.png',
	data: this.qrCodeUrl.slice(22),
	encoding: 'base64',
	success: res => {
        //相册授权
        wx.saveImageToPhotosAlbum({
            filePath: wx.env.USER_DATA_PATH + '/zcbaiy.png',
            success: function(res) {
                wx.showToast({
                    title: '保存成功',
                })
                that.$refs.popups.hidePopup()
            },
            fail: function(err) {
                //授权失败
                if (err.errMsg === "saveImageToPhotosAlbum:fail auth deny" || err.errMsg  === "saveImageToPhotosAlbum:fail authorize no response") {
                    //授权判断
                    wx.authorize({
                        scope: 'scope.writePhotosAlbum',
                        success(re) {
                            console.log(re);
                        },
						fail(err) {
                            wx.showModal({
                                content: '检测到您没打开下载图片功能权限，是否去设置打开？',
                                confirmText: "确认",
								cancelText: "取消",
								success: function(res) {
                                    if (res.confirm) {
                                        //弹出授权框
                                        wx.openSetting({
                                            success: (res) => {}
                                        })
                                    } else {
                                        console.log('用户点击取消')
                                    }
                                }
                            });
                        }
                    })
                }
            }
        })
    },
    fail: err => {
        console.log(err)
    }
})

    说明：

    1. wx.authorize是微信提前向用户发起授权请求。调用后会立刻弹窗询问用户是否同意授权小程序使用某项功能或获取用户的某些数据，但不会实际调用对应接口。如果用户之前已经同意授权，则不会出现弹窗，直接返回成功。
    2. wx.openSetting 是微信调起客户端小程序设置界面，返回用户设置的操作结果
