                                      Vue的基本问题总合

                                      wangshanjian 2020-01-01 vue

                                      # 1.VUE的优点

                                      （1）轻量级框架：只关注视图层，是一个构建数据的视图集合，简单易学。

                                      （2）双向数据绑定：当数据发生变化的时候，视图也就发生变化，当视图发生变化的时候，数据也会跟着同步变化；可以这样说用户在视图上的修改会自动同步到数据模型中去，数据模型也是同样的变化。

                                      （3）组件化：实现了html的封装和重用，在构建单页面应用方面有着独特的优势

                                      （4）视图，数据，结构分离：使数据的更改更为简单，不需要进行逻辑代码的修改，只需要操作数据就能完成相关操作

                                      （5）虚拟DOM：diff算法的使用，新旧虚拟DOM进行比较，减少DOM的操作

                                      # 2.vue中的key的作用和工作原理

                                      key的作用是为了高效的更新虚拟DOM。key给每个节点添加唯一标识，Diff算法就可以正确的识别此节点。vue 在源码patch的过程中通过key可以更加精准判断两个节点是否是同一个，从而避免频繁更新不同元素，使得patch的过程更加高效，减少DOM操作量，提高性能

                                      patch:新旧vnode进行比较,执行updatechildren()，创建、删除或者更新DOM节点/组件实例

                                      例如当data中定义了一个变量a，并且模板中也使用了它，那么这里生成的Watcher就会加入到a的订阅者列表中。当a发生改变时，对应的订阅者收到变动信息，这时候就会触发Watcher的update方法，实际update最后调用的就是在这里声明的updateComponent

                                      # 3.分别简述computed和watch的使用

                                      （1）computed

                                      1.是计算值

                                      2.应用：就是简化template里面{{}}计算和处理props或$emit的传值

                                      3.具有缓存性，页面重新渲染值不变化,计算属性会立即返回之前的计算结果，而不必再次执行函数

                                      （2） watch

                                      1.是观察的动作

                                      2.应用：监听props，$emit或本组件的值执行异步操作

                                      3.无缓存性，页面重新渲染时值不变化也会执行

                                      # 4.$nextTick的使用

                                      在下次dom更新循环结束之后，执行延迟回调。在修改数据之后立即使用这个方法，获得更新后的dom

                                      1.Vue声明周期的created() 钩子函数进行的DOM操作一定要放在Vue.nextTick() 的回调函数中，因为created() 执行的时候DOM实际上并未进行任何渲染，此时进行DOM操作无异于徒劳，所以此处一定要将DOM操作的js代码放进Vue.nextTick()的回调函数中。与之对应的就是mounted 钩子函数，因为该函数执行时所有的DOM挂载和渲染都已完成，此时再钩子函数中进行任何DOM操作都不会有问题。

                                      2.在数据变化后要执行的某个操作，而这个操作需要使用随数据改变而改变的DOM结构的时候，这个操作应该放进Vue.nextTick() 的回调函数中。

                                      # 5.vue组件中data为什么必须是一个函数？

                                      组件中的data写成一个函数，数据以函数返回值的形式定义，这样每次复用组件的时候，都会返回一份新的data，相当于每个组件实例都有自己私有的数据空间，它们只负责各自维护的数据，不会造成混乱。所有的组件实例若共用了一个data，就会造成数据污染。

                                      # 6.双向数据绑定是如何实现的

                                      vue.js 则是采用数据劫持结合发布者-订阅者模式的方式，实现数据与视图的同步。核心Object.defineProperty()来劫持各个属性的setter，getter，在数据变动时发布消息给订阅者，触发相应的监听回调。

                                      # 7.v-if和v-for的优先级

                                      当 v-if与v-for一起使用时，v-for具有比v-if更高的优先级，这意味着v-if将分别重复运行于每个v-for循环中。所以，不推荐v-if和v-for同时使用。 如果v-if和v-for一起用的话，vue中的的会自动提示v-if应该放到外层去

                                      # 8.vue的两个核心点

                                      数据驱动、组件系统

                                      （1）数据驱动：ViewModel，保证数据和视图的一致性

                                      （2）组件系统：应用类UI可以看作全部是由组件树构成的。

                                      # 9.vue更新数组时触发视图更新的方法(数据响应式)

                                      push()、unshift()、pop()、shift()、splice()、sort()、reverse()

                                      # 10.diff算法

                                      • diff是个算法。当新旧虚拟DOM作对比，既产生diff。将变化的地方更新在真实的DOM,同时需要diff高效执行两个DOM树发生的改变，可减低对比过程的时间复杂度。
                                      • 必要性，vue中Watcher力度的降低，每个组件只有一个Watcher与之对应，只有引入diff才能精确的找到变化的地方
                                      • diff过程遵循深度优先，同层比较的策略。比较两个节点的算法，假设首尾节点两两比较，首尾一般会产生相同，可以减少循环。如果没有相同节点才按照通用的遍历方式，查找结束按情况处理剩下的节点。同时借用key更加精准的找到节点，使得patch过程更高效

                                      # 11.vue的组件化

                                      （1）组件化是独立和可复用的代码组织单元，是vue核心特性之一。

                                      （2）组件化开发能够大幅提高应用开发效率、测试性和复用性等

                                      （3）组件可以按使用分类：页面组件（路由），业务组件（登录）、通用组件（按钮）

                                      （4）vue的组件是基于配置的，我们通常写的组件是组件配置而不是组件，框架会生成其构造函数，她们基于VueComponent,扩展于Vue。

                                      （5）Vue常见的组件化技术有：属性prop,自定义属性，插槽，它们主要用于组件通信，扩展等

                                      （6）组件应该是高内聚、低耦合

                                      # 12.vue的设计原理

                                      （1）渐进式JavaScript框架 自底向上逐层应用，关心视图层，不仅容易上手，还便于第三方库的或既有项目的整合

                                      （2）易用性 数据响应式 声明式模板语法 基于配置的组件系统

                                      （3）灵活性 随着应用规模不断扩大，我们可以引入路由、vuex、vue-cli 等工具

                                      （4）高效性 超快的虚拟DOM和diff算法使我们拥有最佳的性能表现

                                      # 13. MVVM的理解

                                      model - view - viewmodel

                                      目的都是为解决Model和View的耦合问题

                                      # 14.vue性能优化

                                      （1）路由懒加载

                                      （2）keep-alive 缓存页面

                                      （3）v-show 处理复用较大代码

                                      （4）图片懒加载 vue-lazyload

                                      （5）第三方插件按需引入

                                      （6）ssr服务器渲染

                                      # 15.vue3.0

                                      （1）更快： 虚拟DOM重写 优化slots的生成 静态树 、静态属性 提升 基于Proxy的响应式系统

                                      （2）更容易维护： TypeScript + 模块化

                                      （3）更友好： 跨平台,使得vue更容易与任何平台使用

                                      # 16.Vue.set()

                                      Vue.set(vm.items, indexOfItem, newValue)

                                      1. vm.items：源数据；
                                      2. indexOfItem：要修改的数据的键
                                      3. newValue：要修改的数据
                                      let a = [
    {name:'张三',age:'20',sex:1},
    {name:'李四',age:'21',sex:0},
    {name:'王五',age:'22',sex:1},
]

Vue.set(a,1,a[1]);  //这时候 a 是源数据， 1 是键， a[1]是修改后的数据
Vue.forceUpdate();  //也可以用这个方法手动触发视图更改

import Vuex from  'vuex'

//使用插件
Vue.use(Vuex)

const store = new  Vuex.Store({
  state: {                                        //变量
    counter:0                                    //this.$store.state.counter
  },
  
  mutations: {                                  //方法
      increment(state){                         //同步函数
          state.counter ++                      ////this.$store.commit('increment')    
      },
      decrement(state){
          state.counter --
      },
      incrementCount(state,n){                    // payload负载
          state.counter += n                      // this.$store.commit('incrementCount'，n)
      }，
      info(state){
          state.info.name = '1564'
      }
  },
  
  actions: {                                     //处理异步函数
      aUpStudent(context,payload){
          return new Promise((resolve,reject) => {
                 setTimeout(() => {
                     context.commit('info')

               })                                //  .then(res => {                               
          })                                     //       console.log('res')                                                             
      }                                          // })

  getters: {                                           //计算属性
     counter(state){
        return state.counter * state.counter           //$store.getters.counter

     ageMore20(state){
        return state.student.filter(s => s.age >15)
      },

        return getter.ageMore20.length
      },
     ageMor(state){
        return age => {
           return state.student.filter(s => s.age >age)
        }
     }
  },
  
  modules: {                                        //用来管理模块独有的特性

})

export  default store

                                      # 18.项目优化

                                      module.exports = {
  chainWebpack: config => {

    // 打包模式
    config.when(process.env.NODE_ENV === 'production', config => {
      config.entry('app').clear().add('./src/main-prod.js')

      // 配置需要使用 CDN 资源加载的第三方依赖包     

     config.set('externals', {
        // 这里写的是需要全局注册的依赖
        'vue': 'Vue',
        依赖名称: 注册名称,
      })

      // 配置 isProd 的值 true 为打包模式
      config.plugin('html').tap(args => {
        args[0].isProd = true
        return args
      })

    })

    // 开发模式
    config.when(process.env.NODE_ENV === 'development', config => {
      config.entry('app').clear().add('./src/main-dev.js')

      // 配置 isProd 的值 false 为开发模式
      config.plugin('html').tap(args => {
        args[0].isProd = false
        return args
      })
      
    })
  }
}
